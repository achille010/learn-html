# When NOT to use Tables

In the early days of the web, tables were used to create page layouts. **This is no longer the case.**

## Why we don't use tables for layout:
1.  **Accessibility**: Screen readers get confused trying to read layout tables.
2.  **SEO**: Search engines prefer semantic HTML like `<header>`, `<main>`, and `<footer>`.
3.  **Maintenance**: Writing nested tables for layout is incredibly hard to manage.
4.  **Responsive Design**: Tables do not shrink and stack well on mobile screens.

## Use CSS Instead!
Today, we use **CSS Flexbox** and **CSS Grid** for page layouts. Join the bridge modules later to see how that works!
