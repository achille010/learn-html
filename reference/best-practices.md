# Best Practices (Dos and Don'ts)

## The DOs
1.  **Use Semantic HTML**: Use the right tag for the right job (`<nav>` for menus, `<button>` for actions).
2.  **Provide Alt Text**: Always describe your images.
3.  **Validate Your Code**: Use tools like the W3C Validator to find errors.
4.  **Keep it Simple**: Don't use 10 tags when 2 will do.
5.  **Separate Concerns**: HTML for structure, CSS for style, JS for behavior.

## The DON'Ts
1.  **Don't use layout tables**: Use CSS (Flexbox/Grid) for positioning.
2.  **Don't use inline styles**: Keep your CSS in separate files.
3.  **Don't skip headings**: Headings are the "skeleton" of your page.
4.  **Don't use "Click Here"**: Use descriptive link text (e.g., "Download our HTML Guide").
5.  **Don't overuse `<div>` and `<span>`**: If a semantic tag exists, use it!
