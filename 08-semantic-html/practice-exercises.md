# Practice Exercises: Semantic HTML

Test your understanding of semantic HTML and how it improves web structure, accessibility, and SEO.

## Quiz

1. What is the main purpose of using semantic HTML tags like `<header>`, `<main>`, and `<footer>`?
   - A) They automatically style the page.
   - B) They provide meaning to the content, helping search engines and screen readers.
   - C) They make the page load faster.
   - D) They are required for JavaScript to work.

2. Which tag should you use for content that is independent and could be distributed on its own (like a blog post or news story)?
   - A) `<section>`
   - B) `<aside>`
   - C) `<article>`
   - D) `<div>`

3. What is the correct way to mark a date for search engines to recognize it?
   - A) `<p>March 2, 2026</p>`
   - B) `<time datetime="2026-03-02">March 2, 2026</time>`
   - C) `<date>March 2, 2026</date>`
   - D) `<span class="date">March 2, 2026</span>`

4. When should you use a `<div>` tag?
   - A) For every block of content.
   - B) Only when there is no semantic alternative and you need a container for styling.
   - C) To define the main navigation.
   - D) Never; `<div>` is deprecated.

## Coding Challenge: Building a Semantic Page

Create an HTML file for a blog post using semantic elements.

Requirements:
- A `<header>` containing the site title and a `<nav>` with a few links.
- A `<main>` element for the primary content.
- Inside `<main>`, use an `<article>` for the blog post.
- Use an `<h1>` for the post title and a `<time>` tag for the publishing date.
- Use a `<section>` for the "Comments" at the bottom of the article.
- Add an `<aside>` for a sidebar (e.g., "About the Author").
- A `<footer>` with copyright information.

## Bonus Challenge

Research the difference between the `<section>` tag and the `<article>` tag. In what scenarios would you nest one inside the other?
