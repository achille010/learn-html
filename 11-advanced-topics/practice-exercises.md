# Practice Exercises: Advanced Topics

Explore modern HTML APIs, custom elements, and the principles of progressive enhancement.

## Quiz

1. Which element provides a scriptable area for drawing 2D and 3D graphics on the fly?
   - A) `<svg>`
   - B) `<canvas>`
   - C) `<draw>`
   - D) `<graphic>`

2. What is the main benefit of using a `<template>` tag?
   - A) It automatically renders content in a specific style.
   - B) It holds HTML that is not rendered on page load but can be instantiated later via JavaScript.
   - C) It improves the SEO of the page.
   - D) It is used to define CSS variables.

3. Which browser API allows you to store small amounts of data locally in the user's browser that persists even after the tab is closed?
   - A) Session Storage
   - B) Cookie API
   - C) Local Storage
   - D) Cache API

4. What is the core idea behind "Progressive Enhancement"?
   - A) Building the most advanced version of a site first and then cutting features for older browsers.
   - B) Ensuring the core functionality works with basic HTML first, then adding layers of CSS and JS.
   - C) Using only the latest JavaScript features.
   - D) Maximizing the use of external libraries.

## Coding Challenge: The Dynamic Web Page

Create an HTML page that demonstrates advanced features.

Requirements:
- A `<canvas>` element with an `id` and a fallback message for browsers that don't support it.
- A `<template>` containing a reusable structure for a "User Card" (e.g., an image, a name, and a bio).
- A button that, when clicked (you can just describe the JS logic), adds a message to the page.
- Using your knowledge of APIs, describe how you would use `localStorage` to save the user's name so it appears the next time they visit.

## Bonus Challenge

Research "Service Workers". What role do they play in creating Progressive Web Apps (PWAs) that work offline?
