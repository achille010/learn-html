# Practice Exercises: Forms

Build interactive and user-friendly forms using various input types and accessibility best practices.

## Quiz

1. Which tag acts as a container for all form elements and defines where the data is sent?
   - A) `<input>`
   - B) `<form>`
   - C) `<submit>`
   - D) `<action>`

2. Why is the `for` attribute on a `<label>` important?
   - A) It sets the color of the label.
   - B) It programmatically links the label to an input's `id`, improving accessibility.
   - C) It makes the label bold.
   - D) It defines the placeholder text.

3. Which input type should you use for sensitive data like a user's password?
   - A) `type="text"`
   - B) `type="hidden"`
   - C) `type="password"`
   - D) `type="secret"`

4. What is the difference between a checkbox and a radio button?
   - A) Checkboxes are square, radio buttons are round.
   - B) Checkboxes allow selecting multiple options; radio buttons allow only one from a group.
   - C) Radio buttons are older than checkboxes.
   - D) There is no difference in behavior.

## Coding Challenge: User Registration Form

Create an HTML form for a user registration page.

Requirements:
- A text input for "Full Name" with a corresponding label.
- An email input with a corresponding label.
- A password input with a corresponding label.
- A "Gender" section using radio buttons (options: Male, Female, Other).
- A "Interests" section using checkboxes (options: Coding, Design, Marketing).
- A `<select>` dropdown for "Country".
- A `<textarea>` for a "Bio".
- A "Submit" button.
- Use `<fieldset>` and `<legend>` to group the personal details.
- Ensure all inputs have proper labels and `id`s.

## Bonus Challenge

Research the `pattern` attribute for the `<input>` tag. How can you use it to ensure a user enters a valid 5-digit zip code?
